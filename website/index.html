Dockerized Static HTML/CSS Website
running with Nginx & published as a Docker Image

Create a simple HTML/CSS website, package it with Docker, run it locally with Nginx, and publish the Docker image to a container registry — delivering a fully containerised, production-deployable web presence from scratch.

Expected Deliverables
✦
Static Website Files Clean HTML & CSS source files forming the site served by Nginx inside the container.
✦
Working Dockerfile A production-grade Dockerfile using nginx:alpine to build a lean, portable image.
✦
Docker Image — Local Built and verified Docker image running the site correctly on localhost port 80.
✦
Published Docker Image Image pushed to Docker Hub or GitHub Container Registry for public distribution.
✦
Setup Guide README with clear build, run, and push commands for the full team to follow.
✦
PR History & Reviews Documented pull requests with peer reviews and approvals showing team collaboration.
Get Running

Docker Commands

Build & Run Locally
# Clone the repo $ git clone https://github.com/zsoftly/project2 $ cd project2   # Build the Docker image $ docker build -t zsoftly-website .   # Run the container on port 8080 $ docker run -d -p 8080:80 --name zsoftly zsoftly-website   # Open → http://localhost:8080
Push to Docker Hub
# Log in to Docker Hub $ docker login   # Tag the image $ docker tag zsoftly-website \     zsoftly/zsoftly-website:v1.0   # Push to registry $ docker push zsoftly/zsoftly-website:v1.0   # Pull anywhere $ docker pull zsoftly/zsoftly-website:v1.0
GitHub Container Registry
# Authenticate with GHCR $ echo $CR_PAT | docker login \     ghcr.io -u USERNAME --password-stdin   # Tag for GHCR $ docker tag zsoftly-website \     ghcr.io/zsoftly/zsoftly-website:v1.0   # Push to GHCR $ docker push ghcr.io/zsoftly/zsoftly-website:v1.0
Manage & Tear Down
# List running containers $ docker ps   # View container logs $ docker logs zsoftly   # Stop and remove container $ docker stop zsoftly && docker rm zsoftly   # Remove image $ docker rmi zsoftly-website
